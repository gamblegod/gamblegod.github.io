mysql repeatable read

REPEATABLE READ （可重复读）保证了在同一个事物中多次读取同样记录的结果是一致的。

可重复读是MySQL的默认事物隔离级别。

# Time 1
# Session 1
mysql> select from tb_tmp;
+———+———+——————+
| name | sex | birth |
+———+———+——————+
| jack | f | 2015-04-24 |
| john | m | 2015-04-24 |
+———+———+——————+
2 rows in set (0.01 sec)
# Session 2
mysql> select from tb_tmp;
+———+———+——————+
| name | sex | birth |
+———+———+——————+
| jack | f | 2015-04-24 |
| john | m | 2015-04-24 |
+———+———+——————+
2 rows in set (0.01 sec)

# Time 2
# Session 1
mysql> start transaction;
Query OK, 0 rows affected (0.00 sec)
# Session 1
mysql> start transaction;
Query OK, 0 rows affected (0.00 sec)

# Time 3
# Session 1
mysql> update tb_tmp set name=’jessica’ where name=’jack’;
Query OK, 1 row affected (0.00 sec)
Rows matched: 1 Changed: 1 Warnings: 0

# Time 4
# Session 1
mysql> select * from tb_tmp;
+————-+———+——————+
| name | sex | birth |
+————-+———+——————+
| jessica | f | 2015-04-24 |
| john | m | 2015-04-24 |
+————-+———+——————+
2 rows in set (0.00 sec)

# Time 5
# Session 2
mysql> select * from tb_tmp;
+———+———+——————+
| name | sex | birth |
+———+———+——————+
| jack | f | 2015-04-24 |
| john | m | 2015-04-24 |
+———+———+——————+
2 rows in set (0.00 sec)

# Time 6
# Session 1
mysql> commit;
Query OK, 0 rows affected (0.01 sec)

# Time 7
# Session 2
mysql> select * from tb_tmp;
+———+———+——————+
| name | sex | birth |
+———+———+——————+
| jack | f | 2015-04-24 |
| john | m | 2015-04-24 |
+———+———+——————+
2 rows in set (0.00 sec)

Session 1已经提交数据，还是看到数据不变，即可以重复读

# Time 8
# Session 2
mysql> commit;
Query OK, 0 rows affected (0.00 sec)

# Time 9
# Session 2
mysql> select * from tb_tmp;
+————-+———+——————+
| name | sex | birth |
+————-+———+——————+
| jessica | f | 2015-04-24 |
| john | m | 2015-04-24 |
+————-+———+——————+
2 rows in set (0.00 sec)

提交事务，看到最新数据
